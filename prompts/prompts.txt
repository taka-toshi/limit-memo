あなたは Webアプリ実装を担当するAIエージェント です。
以下の要求分析・システム分析・内部設計・データ設計に 厳密に従って 実装方針およびコードを生成してください。
設計を変更する判断は行わず、不明点は設計意図を優先して補完してください。

1. アプリ概要
n文字 または nバイトしか入力できない 単一メモ用Webアプリ
PC・スマホ両対応
PWA対応（Chrome install / ホーム画面追加）
同一ユーザーが PCとスマホで同じメモを共有
フロントエンドのみで完結

2. 採用アーキテクチャ（確定事項）

外部クラウドストレージ利用
GitHub Gist または Google Drive を使用
サーバは持たない
同期はクライアント主導
同時編集は想定しない

3. 認証・同期の前提

OAuth 認証を使用
クラウドには 1ユーザー = 1ファイル
Read / Write は以下のタイミングのみ
    起動時
    編集後（必要な場合のみ）
差分同期は行わない
競合時は 後勝ち

4. 内部設計（責務分離）

以下の論理クラス構成を前提とする：
Memo
InputLimiter
LocalStorageRepository
CloudRepository（抽象）
GistRepository（CloudRepository実装）
SyncManager
AuthManager
AppController

責務の移動・統合はなるべく行わないこと。

5. 状態管理（ステートマシン）

以下の状態を持つ：
INIT
LOCAL_ONLY
AUTHENTICATED
SYNCING
SYNCED
OFFLINE

状態遷移は設計済みのものに従う。

6. データ設計（厳守）
6.1 データ構造（JSON）
Root
 ├─ meta
 │   ├─ schemaVersion
 │   ├─ appVersion
 │   └─ createdAt
 ├─ memo
 │   ├─ content
 │   └─ updatedAt
 └─ sync
     ├─ lastSyncedAt
     ├─ lastModifiedBy
     └─ revision

ローカル保存・クラウド保存とも 同一構造
memo 部分は将来暗号化される前提のため、独立して扱うこと

7. 同期ロジック（概念）
revision が大きい方が新
revision が同じ場合は updatedAt を比較
同期成功時に lastSyncedAt を更新
オフライン時はローカルのみ更新

8. 実装時の制約
実装言語：HTML , CSS, JavaScript
セキュリティ最優先
フレームワーク使用は任意（軽量優先）
外部依存は最小限
UIは最小でもよい（機能優先）
暗号化は実装しても良いが、する場合は分離するように。

9. AI agent に期待する出力
実装コード
ファイル構成案
同期処理の流れ説明
PWA対応の要点
OAuth連携時の注意点

10. 禁止事項
サーバサイド実装の提案
同期方式の変更
データ構造の独自拡張
同時編集前提の設計
